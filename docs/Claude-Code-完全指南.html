序言

关于本书

Claude Code 是 Anthropic 官方推出的 AI 编程助手命令行工具，它将 Claude AI 的强大能力直接带入您的终端。与传统的 IDE 插件不同，Claude Code 深度集成了文件系统、Git 工作流和命令行工具，能够真正理解您的项目并进行智能化的代码操作。

本书旨在为开发者提供一份全面、深入的 Claude Code 学习指南，从快速上手到源码分析，帮助您充分利用这一强大的 AI 编程工具。

阅读建议

  • 初学者：建议从第一部分开始，按顺序阅读
  • 有经验用户：可直接跳转到功能详解或实战场景部分
  • 架构师/高级开发者：重点关注深入原理和生态融合部分

认识 Claude Code

了解 Claude Code 的核心特性与优势

什么是 Claude Code

Claude Code 是 Anthropic 官方推出的命令行 AI 编程助手，它具有以下核心特点：

💻
终端原生
直接在命令行中运行，无需 IDE 插件，与 Shell 环境深度集成
🧠
上下文感知
自动理解项目结构，读取相关文件内容，保持对话历史
🔧
工具链集成
内置文件操作工具，Git 工作流支持，命令执行能力
🔌
灵活扩展
支持自定义工具、技能和插件，MCP 协议集成

与其他工具对比

特性 Claude Code Cursor GitHub Copilot Claude.ai
运行环境 终端 IDE IDE Web
项目级理解 ✅ ✅ ❌ ❌
命令执行 ✅ ❌ ❌ ❌
自定义工具 ✅ ❌ ❌ ❌
离线使用 ❌ ❌ ❌ ❌

核心优势

  1. 真正的代码理解：能够阅读整个项目，理解代码结构和依赖关系
  2. 安全的命令执行：在用户确认后执行 Shell 命令
  3. 灵活的扩展性：支持自定义工具、技能和插件
  4. 强大的记忆系统：跨会话保持上下文和项目知识

安装与配置

快速完成 Claude Code 的安装与基础配置

系统要求

  • 操作系统：macOS、Linux、Windows (WSL2)
  • Node.js：v18.0.0 或更高版本
  • 网络：需要访问 Anthropic API

安装步骤

macOS / Linux

# 使用 npm 全局安装
npm install -g @anthropic-ai/claude-code

# 或使用官方安装脚本
curl -fsSL https://claude.ai/install.sh | sh

Windows (WSL2)

# 在 WSL2 环境中安装
npm install -g @anthropic-ai/claude-code

验证安装

claude --version
# 输出: claude-code v1.x.x

认证配置

# 登录 Anthropic 账户
claude login

# 或设置 API Key
export ANTHROPIC_API_KEY=your_api_key_here

基础配置

Claude Code 的配置文件位于 ~/.claude/config.json：

{
  "model": "claude-sonnet-4-6",
  "maxTokens": 4096,
  "temperature": 0.7,
  "autoApprove": false,
  "permissions": {
    "allow": ["Read", "Glob", "Grep"],
    "deny": ["Bash"]
  }
}

理解 Claude Code 的核心架构

会话（Session）

会话是 Claude Code 工作的基本单元，包含：

  • 对话历史：用户与 AI 的交互记录
  • 上下文窗口：当前可用的 Token 空间
  • 工作目录：项目根目录

│  │     Context Window           │    │
│  │  - System Prompt             │    │

钩子系统

在特定事件发生时执行自定义脚本

Hook 机制概述

钩子系统允许您在特定事件发生时执行自定义脚本，实现自动化工作流。

┌─────────────────────────────────────────┐
│           Hook 执行流程                  │
├─────────────────────────────────────────┤
│                                          │
│  事件触发 → PreToolUse → Tool执行        │
│                          ↓               │
│                     PostToolUse          │
│                          ↓               │
│  会话结束 → Stop                         │
│                                          │
└─────────────────────────────────────────┘

Hook 类型

PreToolUse
工具执行前触发，验证参数、阻止危险操作
PostToolUse
工具执行后触发，自动格式化、运行测试
Stop
会话结束时触发，清理临时文件、生成报告

Hook 配置

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "prettier --write \"$CLAUDE_FILE_PATH\"" }]
      }
    ]
  }
}

MCP 协议

模型上下文协议

MCP 架构

┌─────────────┐     ┌─────────────┐
│ MCP Client  │ ←→  │ MCP Server  │
│ (Claude)    │     │ (外部服务)   │
└─────────────┘     └─────────────┘

核心概念:
- Resources: 可读取的数据源
- Tools: 可调用的函数
- Prompts: 预定义的提示词模板

MCP Server 开发

import { Server } from '@modelcontextprotocol/sdk';

const server = new Server({ name: 'my-mcp-server', version: '1.0.0' });

server.tool('read_file', {
  description: '读取文件内容',
  parameters: { type: 'object', properties: { path: { type: 'string' } }, required: ['path'] }
}, async (params) => {
  const content = await fs.readFile(params.path, 'utf-8');
  return { content };
});

server.start();

B. 命令速查表

命令 说明
claude启动交互会话
claude -p "提示词"单次查询
claude --continue继续上次会话
claude --model xxx指定模型
claude --cwd /path指定工作目录
claude --list-sessions列出所有会话

C. 常见问题解答

Q: 如何选择合适的模型？

简单任务使用 Haiku，日常开发使用 Sonnet，复杂架构设计使用 Opus。

Q: 如何保护敏感信息？

使用 .claudeignore 排除敏感文件，配置权限限制。

Q: 会话历史存储在哪里？

存储在 ~/.claude/sessions/ 目录下。
