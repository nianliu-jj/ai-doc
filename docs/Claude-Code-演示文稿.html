📝 如何写好 CLAUDE.md

# 项目名称 ## 技术栈 - 前端: Vue 3 + TypeScript - 后端: Node.js + Express ## 编码规范 - 使用 2 空格缩进 - 函数必须有注释 - 测试覆盖率 >= 80% ## 禁止事项 - ❌ 不要使用 any 类型 - ❌ 不要跳过测试 ## 工作流程 1. 新功能先写测试 2. 代码提交前运行 lint 3. 所有 PR 需要审查

✅四原则

  • 明确具体
  • 提供示例
  • 说明原因
  • 保持更新

📍位置优先级

  • ./.claude/CLAUDE.md (最高)
  • ./CLAUDE.md
  • ~/.claude/CLAUDE.md (最低)

--- name: api-design description: 设计 RESTful API 接口 --- # API 设计技能 ## 参数 - resource: 资源名称（必填） - actions: 操作列表（可选） ## 执行步骤 1. 分析资源属性和关系 2. 设计 URL 路径 3. 定义 HTTP 方法 4. 设计请求/响应格式 ## 使用示例 `/api-design user`
